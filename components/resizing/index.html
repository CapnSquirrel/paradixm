A critical feature of GUI-based operating systems is the ability to change the dimensions of elements.

Typical Appearance

The arrow mouse cursor and resize buttons are the most common tools for element resize across multiple platforms.

Additionally, the most common element that a user resizes is the application window, thus typical appearance of resize components will reference those used for window resize. The resize of different elements require different interaction styles and are explored in the "variants" section.

Arrow Mouse Cursor

The dimensions of a window can be resized by manually changing the dimensions of the window. Manual resize mode is visually represented by a change in the mouse cursor icon's apprearance from a pointer to an arrow. The orientation of the arrow, indicates the direction of resize, whether horizontal, veritical or diagonal. Examples of arrow icons are shown below.

Typical Behavior

Windows are manually resized by clicking a point on along the parameter of the window, and dragging the mouse accordingly to get the window to the desired shape. Windows may alternatively be automatically resized through resize buttons on the window menu, which typically minimize or maximize the window.

Events

Standby - Hover (Reveal transformation options) - Click - (for direct) Choose Output State - Release Click - Resized - Standby

The most relevent resize event is the resize event. A resize event is triggered when the size of the window has been changed. While maximizing or minimizing a window, a minimize or maximize event may also be triggered.

State Diagram

A window is typically in a standby phase before being resized. Once the mouse starts dragging the edge or corner, the window enters the resizing state. Once finished, the window reaches the resized state.

    Priority Metrics

    Resize is a critical and commonly used aspect of any GUI-based operating system. When evaluating this interaction component against Ben Shneiderman's five usability metrics outlined in his book Designing the User Interace, the most important metrics for resize are learnability, efficiency, errors, while less important are: memorability and satisfaction. Since it is a critical aspect of any GUI based operating system, most of the priority metrics are vital in this component's design analysis. Users who intend to resize a window must have no confusion as to how to accomplish the task. In the physical world, the natural way of unfolding or unrolling a piece of paper is to pull from either the sides or the corners. Resizing follows this natural way of thinking by allowing the user to increase or decrease the size from the corners and sides of the window, making the process simple to learn. Also, since resizing follows the natural mental model, the way in which the user resizes windows is easily memorable.

    Since window resizing corresponds to either mouse input or window menu input, the response is immediate. Since window resizing is easily learnable and highly memorable, it is unlikely that users commit errors when attempting the task.

    Due to its simplicity and utilitarian nature, resizing does very little in the way of providing satisfaction to the user. However, this does not detract from the overall good design of window resizing.

    Key Characteristics

    Key characteristics of resizing include the changing of the mouse icon from a pointer to arrows along with the visible change in window size.

    Feedback

    For resizing, the window visibly grows larger or shrinks according to user input. In older operating systems, a rectangle is shown to represent the size the window will change to when the user stops dragging the mouse.
